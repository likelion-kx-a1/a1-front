# 요구사항

## 1. 문서 정보

### 목적

영상 제작을 위한 생성 자산 서비스의 요구사항을 정의한다. 사용자는 텍스트로 기준 이미지(Key Image)를
생성하고 영상 제작에 재사용하며, 생성된 자산은 라이브러리에서 검색·재사용·변형된다.
즉 "생성 → 관리 → 재사용 → 변형"으로 이어지는 제작 흐름을 하나의 Workflow로 제공한다.

## 주요 기능 범위

- 이미지 생성
  - Key Image 생성 및 저장
- 영상 생성
  - Text -> Video
- 이미지 기반 영상 생성
  - Image + Prompt → Video
- 역프롬프트
  - 프롬프트 추출 및 재생성
- 라이브러리
  - 생성 자산 저장 및 관리
  - AI 메타데이터 생성 및 자동 태깅
  - 생성 자산 검색
- 이미지 수정 및 변형 생성
  - 변형 결과물 생성


## 2. 시스템 개요

### 해결하려는 문제

- 방송 제작자는 이미지 생성·영상 생성·프롬프트 생성·이미지 분석 등 여러 AI 서비스를 오가며 작업한다.
- 생성 결과를 다운로드 → 재업로드 → 프롬프트 재작성하는 과정을 반복해야 한다.
- 도구가 분산되어 있어 작업 흐름이 끊기고 반복 작업이 늘어난다.

### 핵심 가치

- 도구를 옮겨다니지 않는 하나의 창작 흐름
- 한 번 만든 이미지가 영상까지 이어지는 재사용
- 저장하고 검색하는 것만으로 쌓이는 나만의 자산

## 3. 기능 요구 사항

### 랜딩

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
| LP1 | 1순위 | 사용자는 첫 접속 시 작업(랜딩) 페이지에서 에셋 예제와 템플릿을 조회할 수 있다. |
| LP2 | 1순위 | 사용자는 좌측 사이드바에서 홈, 라이브러리, 이미지/비디오/스토리보드 생성 메뉴로 이동할 수 있다. |
| LP3 | 1순위 | 사용자는 좌측 하단에서 계정 상태를 확인할 수 있다. |

### 인증

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
| AU1 | 1순위 | 사용자는 이메일·비밀번호로 로그인할 수 있다. |
| AU2 | 1순위 | 사용자는 소셜 계정으로 로그인할 수 있다. |
| AU3 | 1순위 | 비로그인 사용자가 생성 기능을 실행하면 로그인 페이지로 유도된다. |

### 이미지 생성

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
| I1  | 1순위 | 사용자는 프롬프트 입력창에 텍스트를 입력해 Key Image를 생성할 수 있다. |
| I2  | 1순위 | 사용자는 배경 이미지를 생성할 수 있다. |
| I3  | 1순위 | 사용자는 캐릭터 이미지를 생성할 수 있다. |
| I4  | 1순위 | 사용자는 기존 이미지를 수정·변형해 새로운 결과물을 생성할 수 있다. |
| I5  | 1순위 | 사용자는 생성한 이미지를 다운로드할 수 있다. |
| I6  | 1순위 | 사용자는 생성한 이미지를 라이브러리에 저장할 수 있다. |

### 영상 생성

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
| V1  | 1순위 | 사용자는 텍스트 프롬프트를 입력해 영상(Text→Video)을 생성할 수 있다. |
| V2  | 1순위 | 사용자는 이미지와 프롬프트를 함께 입력해 영상(Image+Prompt→Video)을 생성할 수 있다. |
| V3  | 1순위 | 사용자는 생성한 영상을 다운로드할 수 있다. |
| V4  | 1순위 | 사용자는 생성한 영상을 라이브러리에 저장할 수 있다. |
| V5  | 2순위 | 사용자는 스토리보드를 불러와 영상 생성에 활용할 수 있다. |


### 역프롬프트

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
| R1  | 1순위 | 사용자는 이미지에서 프롬프트를 추출(역프롬프트)할 수 있다. |
| R2  | 2순위 | 사용자는 추출된 프롬프트를 편집해 결과물을 재생성할 수 있다. |

### 라이브러리

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
| L1  | 1순위 | 사용자는 프로젝트별로 생성 자산을 저장·관리할 수 있다. |
| L2  | 1순위 | 사용자는 전체·배경·캐릭터 등 탭으로 에셋을 필터링해 탐색할 수 있다. |
| L3  | 1순위 | 사용자는 키워드로 생성 자산을 검색할 수 있다. |
| L4  | 1순위 | 사용자는 프롬프트 히스토리를 탐색하고 해당 에셋을 다운로드할 수 있다. |
| L5  | 2순위 | 시스템은 생성 자산을 AI로 분석해 설명 정보와 검색용 태그를 자동으로 생성한다. |
| L6  | 2순위 | 사용자는 라이브러리에서 에셋을 다운로드하거나 역프롬프트를 실행할 수 있다. |

| M3  | 2순위 | 사용자는 자신의 토큰 사용량을 조회할 수 있다. |
| M4  | 2순위 | 사용자는 토큰 사용량 한도를 설정할 수 있다. |

| ID  | 우선순위 | 요구사항 |
| --- | -------- | -------- |
